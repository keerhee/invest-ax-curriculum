Workflow

워크플로우 흐름 (질의)

🔴 Trigger→ Webhook — 자연어 질의 + ticker 필터
🟣 Embed→ OpenAI 질의 임베딩
🟢 Vector→ Supabase top-20 cosine search (필터: ticker)
🟢 BM25→ Supabase ts_rank — 키워드 매칭 top-20 (병렬)
⚫ Merge→ Code 노드 — 두 결과 dedup, 합집합 ~30건
🔵 Rerank→ Cohere Rerank — top 5 재순위
🟣 Answer→ Claude Sonnet — top-5를 컨텍스트로 인용 답변
🟢 Reply→ Slack — 답변 + [출처1, 출처2] 링크
Schema

Supabase 테이블 (W5 확장)

create table disclosures (
  id bigserial primary key,
  ticker text not null,
  doc_type text,           -- '사업보고서'|'분기'|'주요공시'
  doc_year int,
  page int,
  section text,
  content text,
  embedding vector(1536),
  fts tsvector              -- 한국어 BM25용
);

-- 인덱스
create index on disclosures using ivfflat (embedding vector_cosine_ops);
create index on disclosures using gin (fts);

-- Trigger로 fts 자동 생성
create function disclosures_fts_trigger() returns trigger as $$
begin
  new.fts := to_tsvector('simple', new.content);
  return new;
end $$ language plpgsql;

create trigger disclosures_fts_update before insert or update
on disclosures for each row execute function disclosures_fts_trigger();

Qdrant Native Hybrid + 금융 응용 실습

본 주차 본편은 Supabase pgvector + ts_rank로 hybrid를 구현했지만, Qdrant 1.10+는 동일한 흐름을 한 번의 API 호출로 끝내는 native hybrid를 제공합니다. 사업보고서·공시· 애널리스트 리포트 검색에 그대로 적용해봅시다.

왜 Qdrant Hybrid?

1. Supabase 방식 vs Qdrant Native 비교

항목Supabase pgvector + ts_rankQdrant Native Hybrid
호출 횟수2회 (vector + BM25 따로)1회 (Query API prefetch)
RRF 결합n8n Code 노드에서 수동서버 측 자동
Sparse 모델Postgres 한국어 형태소 한계BM25 / SPLADE 선택
한국어 처리tsvector 한계 (조사 처리)fastembed 다국어 가능
스케일500MB 무료, 단일 PG1GB 무료, 분산 가능
학습 곡선SQL 익숙하면 쉬움Named vector 개념 필요
Architecture

2. Named Vectors — 하나의 컬렉션, 두 개의 벡터

collection: "invest_disclosures"
  named vectors:
    - name: "dense"
      size: 1536
      distance: Cosine          # OpenAI text-embedding-3-small
    - name: "sparse"
      modifier: idf              # BM25 내장 IDF
      # sparse vector indices/values 형식
  payload:
    - ticker, doc_type, doc_year, page, section, content

임베딩 단계에서 한 청크당 dense + sparse 두 벡터를 동시에 저장. 검색은 둘을 한 호출로.

Workflow

3. n8n 워크플로우 흐름

(A) 임베딩 (PDF 업로드 시 1회)

🔴 Trigger→ Drive Trigger — 사업보고서 PDF 업로드
🔵 Extract→ PDF → 텍스트 → 1000자 청크 (overlap 200)
🟣 Dense→ OpenAI text-embedding-3-small (1536d)
🟣 Sparse→ FastEmbed BM25 (서버 함수 또는 HTTP)
🟢 Upsert→ Qdrant /points (vectors: {dense, sparse} 동시)

(B) 검색 (질의)

🔴 Trigger→ Webhook — 자연어 질의
🟣 Embed × 2→ Dense (OpenAI) + Sparse (BM25) 동시 생성
🟢 Hybrid Q→ Qdrant /points/query — prefetch [dense, sparse] + RRF fusion 한 번에
🟣 Answer→ Claude — top-K로 인용 답변
🟢 Reply→ Slack / Webhook 응답
API

4. Qdrant Hybrid Query API (한 번의 호출)

컬렉션 생성

PUT https://YOUR-CLUSTER.qdrant.tech/collections/invest_disclosures
{
  "vectors": {
    "dense": { "size": 1536, "distance": "Cosine" }
  },
  "sparse_vectors": {
    "sparse": { "modifier": "idf" }
  }
}

임베딩 upsert

PUT /collections/invest_disclosures/points
{
  "points": [{
    "id": 1,
    "vector": {
      "dense":  [0.12, -0.05, ...1536개],
      "sparse": { "indices": [42, 1024, ...], "values": [0.7, 0.3, ...] }
    },
    "payload": {
      "ticker": "005930",
      "doc_type": "사업보고서", "doc_year": 2024, "page": 187,
      "section": "연구개발 활동", "content": "..."
    }
  }]
}

Hybrid Search (RRF 한 번에)

POST /collections/invest_disclosures/points/query
{
  "prefetch": [
    { "query": [0.12, -0.05, ...], "using": "dense",  "limit": 20 },
    { "query": {"indices":[...], "values":[...]}, "using": "sparse", "limit": 20 }
  ],
  "query": { "fusion": "rrf" },
  "limit": 5,
  "with_payload": true,
  "filter": { "must": [{"key": "ticker", "match": {"value": "005930"}}] }
}

응답은 두 검색의 RRF 결합 top-5. "fusion": "dbsf"로 distribution-based score fusion도 선택 가능.

금융 응용

5. 어떤 질문에 어느 쪽이 강한가

금융 질의Dense만BM25만Hybrid
"삼성전자 24년 R&D 투자액은?" (정확 숫자)△ 비슷한 분기 섞임✅ 정확 페이지✅
"AI 메모리 사이클 전망" (의미)✅ 동의어 잡음❌ 키워드 한정✅
"HBM3E 단독공급" (고유명사)△ 일반화됨✅ 정확✅
"지배구조 ESG 리스크" (개념)✅ 의미 검색△ 표현 다양✅
"공시번호 20240315000123" (식별자)❌✅ exact✅
핵심 인사이트 — IR/공시는 숫자·고유명사·식별자가 많아 BM25가 강하고, 전략 변화·리스크·시장 전망은 의미 검색이 강합니다. Hybrid가 둘 다 잡습니다.
