4 Risk Guards

4단계 Risk Guards

가드체크위반 시
1. 단일 종목 비중이번 주문 후 단일 종목 비중 ≤ 10%주문 취소 + Slack 경고
2. 일일 체결 횟수당일 체결 ≤ 5건당일 추가 주문 차단
3. 변동성 임계20일 변동성 ≤ 5%주문 수량 절반
4. DDL (Daily Drawdown Limit)당일 평가손익 ≥ -2%매수 전면 중단 (안전모드)

4가지 모두 통과한 시그널만 Bracket Order로 발주. 하나라도 실패 시 사유와 함께 Slack 알림.

Bracket Order

Bracket Order 구조

한 번의 주문에 3개 다리가 동시 등록:
1. Entry — 시장가 또는 지정가 매수/매도
2. Take Profit — +5% 수익 시 자동 매도
3. Stop Loss  — -2% 손실 시 자동 매도

→ 진입 직후부터 보호선이 살아있음 → 인간 개입 없이도 손실 한정
Workflow

워크플로우 흐름

🔴 Trigger→ W7 Orchestrator 결과 — verdict=BUY_STRONG/SELL_STRONG만
🔵 Position→ Alpaca GET /v2/positions — 현재 포지션 조회
🔵 Account→ Alpaca GET /v2/account — 잔고 + 일일 손익 조회
⚫ Risk × 4→ Code — 4 가드 순차 체크, 첫 실패 시 short-circuit
🔵 Bracket→ Alpaca POST /v2/orders — order_class=bracket
🟢 Log+Notify→ Sheets 주문이력 + Slack 발주 알림 (모의)

Risk Guards Code

Risk Guards JavaScript 골격

// 입력: {ticker, side, qty, price}, position[], account
// 출력: {pass: true|false, failed_guard?, reason?}

const orderValue = qty * price;
const portfolioValue = +account.portfolio_value;

// Guard 1: 단일 종목 비중 ≤ 10%
const myPosVal = positions.filter(p => p.symbol===ticker)
                          .reduce((s,p) => s + +p.market_value, 0);
const newConcentration = (myPosVal + orderValue) / portfolioValue;
if (newConcentration > 0.10) {
  return {pass:false, failed_guard:1, reason:`single ${(newConcentration*100).toFixed(1)}%`};
}

// Guard 2: 일일 체결 횟수 ≤ 5
const todayFills = await getAlpacaFillsToday(); // helper
if (todayFills.length >= 5) {
  return {pass:false, failed_guard:2, reason:`today ${todayFills.length} fills`};
}

// Guard 3: 변동성 ≤ 5%
const vol20 = await get20DayVolatility(ticker); // helper from W2
if (vol20 > 0.05) {
  // 수량 절반으로
  qty = Math.floor(qty/2);
}

// Guard 4: DDL ≥ -2%
const dailyPnL = (+account.equity - +account.last_equity) / +account.last_equity;
if (dailyPnL < -0.02 && side === 'buy') {
  return {pass:false, failed_guard:4, reason:`DDL ${(dailyPnL*100).toFixed(1)}%`};
}

return {pass:true, qty};
